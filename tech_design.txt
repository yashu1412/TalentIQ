4. Low-Level Design (LLD)
4.1 Resume Processing Pipeline
End-to-End Flow
User uploads PDF via POST /resumes/upload
FastAPI saves file to S3, inserts resumes row with parse_status='pending'
Celery task resume_parse_task(resume_id) pushed to Redis high-priority queue
Worker fetches file from S3 URL
PyMuPDF (fitz) opens file, joins page text into raw_text
Section detector (regex) splits text into SKILLS / EXPERIENCE / EDUCATION / PROJECTS
LangChain extraction chain sends sections to GPT-4o → returns structured ResumeProfile
ATS scorer computes keyword hit-rate against 500-word tech vocabulary → ats_score
Embedder: sliding window chunker (512 tokens, 64 overlap) → OpenAI embeddings → upsert to document_embeddings
resumes row updated: parse_status='done', parsed_json, ats_score
Frontend polls GET /resumes/{id} until parse_status='done'

copilot/memory.py — Vector Memory Retrieval
def retrieve_context(user_id: UUID, query: str, k: int = 5) -> list[str]:
    query_emb = openai.embed(query)
    return pgvector_search(
        user_id=user_id,
        embedding=query_emb,
        doc_types=['chat', 'resume', 'note'],
        k=k,
        metric='cosine'
    )

live_room/executor.py — Piston Integration
async def run_code(language: str, code: str, stdin: str = '') -> ExecutionResult:
    payload = {
        "language": language,
        "version": LANGUAGE_VERSIONS[language],  # pinned per-language
        "files": [{"content": code}],
        "stdin": stdin,
        "run_timeout": 5000,    # 5s wall-clock limit
        "compile_timeout": 10000
    }
    resp = await httpx.post(PISTON_URL + '/api/v2/execute', json=payload)
    return ExecutionResult(**resp.json()['run'])

5. API Contracts
Base URL: https://api.talentiq.dev/v1
Auth: All endpoints (except /auth/webhook) require  Authorization: Bearer <Clerk JWT>
5.1 Auth / Profile
POST /auth/webhook
Description: Clerk webhook — sync new/updated user into PostgreSQL
Auth: Svix webhook signature verification (no JWT)
Body: Clerk UserCreated / UserUpdated event payload (JSON)
Response 200: { "ok": true }
GET /me
Description: Return authenticated user + profile
Response 200: { id, email, full_name, role, profile: { target_roles, experience_level, skills_summary, bio } }
PATCH /me/profile
Description: Update user profile fields
Body: { target_roles?: string[], experience_level?: string, preferred_locations?: string[], bio?: string }
Response 200: Updated UserProfile object
5.2 Resume
POST /resumes/upload
Description: Upload PDF resume — file stored in S3, parse job enqueued
Content-Type: multipart/form-data
Body field: file: PDF binary (max 5 MB)
Response 202: { resume_id: UUID, status: "pending" }
GET /resumes
Description: List all resumes for the authenticated user
Response 200: Resume[] — [{ id, file_name, ats_score, parse_status, created_at }]
GET /resumes/{id}
Description: Full parsed resume detail — poll until parse_status = 'done'
Response 200: { id, raw_text, parsed_json, ats_score, quality_score, current_version, parse_status }
POST /resumes/{id}/improve
Description: AI rewrite of a specific resume section against a target JD
Body: { section: "experience" | "skills" | "projects", target_jd_id?: UUID }
Response 200: { original: string, improved: string, version_id: UUID }
GET /resumes/{id}/versions
Description: List version history for a resume
Response 200: ResumeVersion[] — [{ id, version_number, diff_summary, created_at }]
5.3 Jobs
POST /jobs/analyze
Description: Parse a raw job description (text or URL) — async
Body: { jd_text?: string, url?: string }  (one required)

GET /jobs/{id}
Response 200: { id, title, company, jd_text, parsed_json: { must_have[], nice_to_have[], years_required } }
